Module 1

Web Programming Basics

Unit 1: Definitions of Web Programming, the Internet and World Wide Web (WWW)

This module begins by introducing some theoretical concepts of the Internet and the Web. It explains the importance and the need for web programming.

What is Web Programming?

Web programming is the process of writing codes that will eventually create a website. Texts and multimedia contents can be manipulated by a web designer to produce an elegant page that is seen on a web browser.

Importance of Web Programming
  • Global Reach: The Internet and the web allow businesses to reach a worldwide audience with their products and services.
  • Cost-Effectiveness: Web development enables businesses to reduce costs by automating processes and improving operational metrics.

What is the Internet?

The Internet is a global communication system consisting of hardware and software infrastructures that provide connectivity between remote computers.

What is the Web?

The Web is a collection of interconnected documents and other resources, linked by hyperlinks and universal resource locators (URLs).

How to Connect to the Internet

  1. Wireless Broadband (Wi-Fi): Connection via wireless transmission configurations setup on portable parameters.
  2. Ethernet Cable: Hardware data streaming execution mapping using Cat5 or Cat6 wiring connections.
  3. Dial-up Connection: Legacy structural link configurations operating over phone modem connections.

Module 2

Hypertext Markup Language (HTML)

Unit 1: HTML Tags and Attributes

HTML files are compiled using structural element components known as tags. Most tags have an opening marker and a closing marker. Attributes provide extra configuration values to these elements inside the opening block.

  • <h1> to <h6> - Section Heading definitions.
  • <p> - Core paragraph block formatting element.
  • <a href="..."> - Hyperlink resource mapping property.

Unit 2: Structure of an HTML Page

Every standard compliant document follows a strict framework deployment configuration. The foundation markup maps out precisely like this template execution:

  • DOCTYPE declaration: Tells browsers to read standard HTML5 rules.
  • HTML Root element: Wraps up all coding entries securely.
  • Head container: Holds meta configuration attributes, styles, and tab titles.
  • Body block: Displays visual render layouts to end-point site visitors.

Module 3

Cascading Style Sheets (CSS)

Unit 1: Basics of CSS

CSS dictates the visual appearance, color maps, and space parameters of structural layouts. It links configuration properties to target selectors utilizing structured syntax blocks:

  • Selector: Points to the specific HTML tag to style (e.g., body).
  • Declaration: Declares properties and values enclosed inside curly braces.

Unit 2: Adding Style to HTML

Visual styling definitions can be appended to an application via three distinct methods:

  1. Inline Style: Set directly inside tag blocks using the style="..." property attribute.
  2. Internal Stylesheet: Declared globally within the head using <style> element blocks.

Unit 3: CSS Text, Font, and Colors

Typography layouts manage reading layouts via specific attributes:

  • color - Modifies the foreground text hue (e.g., #c084fc).
  • font-family - Selects the type face structure (e.g., 'Segoe UI', system-ui).
  • text-align - Handles structural alignments like center or justify positioning rules.
